Task 1 - Creating a Virtual Private Cloud (VPC)

A Virtual Private Cloud (VPC) is a logically isolated section of the AWS Cloud where you can launch AWS resources in a virtual network that you define. It provides you with complete control over your virtual networking environment, including selection of your own IP address range, creation of subnets, and configuration of route tables and network gateways.

The benefits of using a VPC include:

  • Enhanced security: VPCs allow you to isolate your resources and control network access.
  • Customizable network configuration: You can design your network architecture to suit your specific needs.
  • Scalability: VPCs can easily grow with your requirements, allowing you to add or remove resources as needed.
  • Cost-effectiveness: You only pay for the resources you use within your VPC.
  • Improved performance: VPCs enable you to optimize network traffic and reduce latency.

  1. One the VPC Management page, select Create VPC.
  1. On the Create VPC window, enable the VPC only radio button.

After a VPC has been created, additional IPv4 CIDR or IPv6 CIDR IP ranges can be added depending on the need for additional IP addresses for applications or databases.

Task 2 - Create Subnet

A subnet is a segmented portion of a VPC's IP address range. Subnets serve several important functions within a VPC:

  • Network segmentation: Subnets allow you to divide your VPC into smaller, more manageable network segments.
  • Improved security: By grouping resources into different subnets, you can control network access and apply security policies more granularly.
  • Availability Zone distribution: Subnets are tied to specific Availability Zones, enabling you to spread your resources across multiple AZs for high availability and fault tolerance.
  • Traffic control: Subnets help in routing traffic efficiently within the VPC and to external networks.
  • Resource organization: You can organize your AWS resources logically by placing them in appropriate subnets based on their function or security requirements.

Creating subnets within your VPC is an essential step in designing a robust and secure network architecture in AWS.
