Overview

The Amazon Web Services (AWS) cloud platform offers scalable cloud storage solutions. These solutions enable companies to store and share data in the cloud, replacing traditional on-premises file servers. This approach minimizes a company's server footprint and reduces maintenance needs.

One such solution is Amazon S3 (Simple Storage Service), which allows users to access data from anywhere, at any time.

In this task, we will create an S3 storage solution.

Task 1: Create an S3 Storage Solution

  1. One the AWS Management Console, select Services, then scroll down to Storage, and then select S3. Alternatively, search for S3.
  1. From the Amazon S3 management page, select Create Bucket.
  1. Name the bucket. Note that the bucket name must be unique.
  1. I’m going to leave all default options. Note that server-side encryption is automatically applied to objects store in the bucket.

    Scroll down and select Create Bucket.

Note: Buckets in Amazon S3 are used as containers to store files. It is also used to create permissions sets so only authorized users will access the data in the bucket. To organize the data in the bucket, folders can be created with a specific naming convention.

  1. Create a folder in the new bucket. Name of the folder can’t contain “/”. Once named, click create.
  1. Next, we will upload a document of your choosing to the newly created folder. Select the folder, and then click upload.

Amazon S3 charges for storing objects based on several factors:

  • Storage class: Different storage classes (e.g., Standard, Intelligent-Tiering, Glacier) have different pricing tiers.
  • Amount of data stored: Charges are based on the volume of data stored in gigabytes (GB) per month.
  • Duration of storage: Prices are pro-rated for the number of days objects are stored.
  • Data transfer: Charges apply for data transferred out of S3 to the internet or other AWS regions.
  • Requests: There are small fees for the number and type of requests made to your objects.

Amazon provides a pricing calculator to estimate costs based on your specific usage patterns. It's important to choose the appropriate storage class and implement lifecycle policies to optimize costs.

  1. Click Add Files. Upload your file.

Note that clicking the Properties dropdown allows you to choose the storage tier.

  1. Verify that the file was successfully uploaded.
  1. To view the image, click the file in your folder, then click Open.

Note that the image is not publicly available from S3 by default due to Amazon's security-first approach. Here's why:

  • Default private access: When you create an S3 bucket, all objects within it are private by default. This means only the AWS account owner has access to the objects.
  • Security best practices: Amazon implements this as a security measure to prevent accidental exposure of potentially sensitive data.
  • Access control: To make an object public, you need to explicitly change its permissions or the bucket's policy. This gives you granular control over what is shared and what remains private.
  • Compliance requirements: Many organizations have strict data privacy regulations. Keeping objects private by default helps in maintaining compliance.

If you want to make an object publicly accessible, you would need to modify its permissions or create a bucket policy that allows public read access. However, it's crucial to carefully consider the implications of making any S3 object public, especially if it contains sensitive information.
