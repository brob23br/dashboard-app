AWS Regions, Availability Zones, and Global Edge Networks

Tags		AWSAWS Cloud PractitionerRDSS3VPC
Subjects		Percipio Lab

AWS Regions: Geographically distinct locations that provide physical infrastructure to deploy applications and services.

Availability Zones (AZs): Multiple isolated locations within each AWS Region, ensuring redundancy and reliability for applications.

Global Edge Networks: A network of data centers located globally to deliver content with low latency and high transfer speeds.

Task 1: Configure Virtual Private Cloud and Explore Availability Zones

AWS Virtual Private Cloud (VPC): A logically isolated network in the AWS cloud that enables you to launch and manage AWS resources securely. VPC allows customization of network configurations, and control over access and data flow within your cloud environment.

  1. On the AWS Management Console, select Services. On the left pane, find Networking and Content Delivery and then select VPC.
  1. Once on the VPC dashboard, select Create VPC.
  1. For VPC settings changes from default, I will be naming the VPC Demo1a, changing the IPv4 CIDR block to 172.16.0.0/16, and selecting NAT gateways 1 per AZ.
  • Availability Zones are equipped with redundant power, networking, and connectivity, allowing them to provide high availability and fault tolerance for applications. Each AZ is physically separated from others within an AWS Region, reducing the risk of simultaneous failures.
  1. VPC successfully created.

Task 2: Configure S3 Storage

Amazon S3 (Simple Storage Service): Amazon S3 is a scalable object storage service that provides secure, durable, and highly available storage for a wide range of data. It allows users to store and retrieve any amount of data from anywhere on the web. S3 is designed to deliver 99.999999999% (11 9's) of durability, ensuring that data is protected against failures, errors, and disasters. It supports various storage classes to optimize cost and performance based on data access needs, making it ideal for use cases such as backup and restore, archival, big data analytics, and content distribution.

  1. On the console management page, go to Services, Storage and then S3.
  1. On the Amazon S3 dashboard, select Create bucket.
  • An S3 bucket is essentially a storage container in Amazon's Simple Storage Service (S3) where users can store, organize, and manage their data. It provides a scalable, secure, and durable storage solution for a wide range of data types, accessible from anywhere over the internet. S3 buckets are highly versatile, supporting various use cases including data backup, content distribution, and big data analytics.
  • Note that the bucket name must be unique within the global namespace.
  1. I will be naming the bucket Demo1abucket. All other configuration options will remain on default.
  1. Verify bucket successfully created in the correct region.
  • An AWS Region is a physical location spread across the globe where Amazon Web Services clusters data centers. Each region consists of multiple Availability Zones, which are isolated and physically separate from each other, providing high availability and fault tolerance for applications. By selecting the appropriate AWS Region, users can optimize latency, ensure data sovereignty compliance, and improve disaster recovery capabilities.

Task 3: Explore RDS and Availability Zones

Amazon Relational Database Service (RDS): Amazon RDS is a managed relational database service that simplifies the setup, operation, and scaling of a relational database in the cloud. It offers support for several popular database engines, including MySQL, PostgreSQL, MariaDB, Oracle, and Microsoft SQL Server. With RDS, you can deploy a highly available and scalable database solution with automated backups, software patching, and maintenance, allowing you to focus on application development rather than database management. RDS also provides security features like encryption and network isolation to protect your data.

  1. Navigate to Services, Database, and then RDS.
  1. On the dashboard, scroll down to find the Create Database button.
  1. Scroll down the page until you see the Availability and durability section and select Info for more information about Multi-AZ DB Cluster, Multi-AZ DB instance, and Single DB instance. Review the different AZ deployments available for RDS.
