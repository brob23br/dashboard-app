Mapping a network drive in Linux Mint

Tags		linuxmintstorageterminal

This lab will guide you through the process of mapping a network drive in Linux Mint using the command line. Here's a brief overview of the steps:

  1. Update the system and install necessary packages
  1. Create a mount point directory for the network drive
  1. Use the mount command to connect to the network share
  1. Verify the successful mounting of the network drive

By following these steps, you'll learn how to efficiently map and access network drives in Linux Mint, enhancing your file management capabilities in a networked environment.

  1. Install Required Package:
    • Open a terminal and run the following command to ensure that cifs-utils is installed:
      sudo apt update
sudo apt install cifs-utils
  1. Create a Mount Point:
    • Create a directory where you want to mount the network drive (e.g., /mnt/shared):
      sudo mkdir /mnt/shared
  1. Mount the Network Drive:
    • Use the following command to mount the network drive: (I will be running the command without the password flag. It will prompt you for the password afterwards.)
      sudo mount -t cifs -o username=your_username,password=your_password //server_ip_or_name/shared_folder /mnt/shared
    • Replace:
      • your_username with your username for the network share.
      • your_password with your password for the network share.
      • server_ip_or_name with the IP or hostname of the server.
      • shared_folder with the name of the shared folder.
  1. Verify the Mount:
    • To check if the drive is mounted, run:
      df -h
    • You should see the network share listed among the mounted file systems.
