Creating an EC2 Virtual Machine using AWS Management Console

Tags		AWSAWS Cloud PractitionerEC2RDPWindows
Subjects		Precipio Labs

Overview

Amazon Web Services (AWS) enables users to create virtual machines on its public cloud infrastructure. Known as Infrastructure as a Service (IaaS), it offers a highly scalable environment for provisioning virtual machines across various AWS regions. The Elastic Compute Cloud (EC2) service is responsible for creating these virtual devices.

To facilitate communication between virtual devices on the AWS Cloud Platform, a Virtual Private Cloud (VPC) must be established.

In this exercise, we'll create a Virtual Private Cloud using the Networking & Content Delivery Service, and set up a virtual machine using EC2 Services in the AWS Management Console.

Task 1 - Create a VPC

Virtual Private Cloud

An AWS Virtual Private Cloud (VPC) is a logically isolated section of the AWS cloud where you can launch AWS resources in a virtual network that you define. It provides control over your virtual networking environment, including selection of your own IP address range, creation of subnets, and configuration of route tables and network gateways.

Key features of AWS VPC include:

  • Complete control over your virtual networking environment
  • Ability to secure your AWS resources
  • Multiple connectivity options to connect your VPC to other networks
  • Customizable network configuration
  • Scalability to support your growing infrastructure needs

VPCs are fundamental to network security and resource isolation in AWS, allowing you to build a secure and scalable cloud infrastructure.

  1. First a VPC must be made. From Services, navigate to Networking & Content Delivery and then select VPC.
  1. Select Create VPC.
  1. Select VPC and more. Complete the following fields:
Name tag auto-generation: projectvpc321
IPv4 CIDR block:   172.16.0.0/16
Number of Availability Zones (AZs): 1
NAT gateways ($): In 1 AZ
VPC endpoints: S3 Gateway
  1. Click Create VPC. It may take some time for the VPC to be created.
  1. Once created, return to the VPC dashboard and select Subnets in the left pane.
  1. You should see that two subnets were created.

When creating a VPC using the "VPC and more" option in AWS, two subnets are typically created by default for the following reasons:

  • High Availability: AWS recommends using multiple subnets across different Availability Zones (AZs) for redundancy and fault tolerance. By creating two subnets, AWS ensures that your resources can be distributed across multiple AZs.
  • Public and Private Subnets: Often, one subnet is configured as public (with internet access) and the other as private (for internal resources). This setup allows for a more secure network architecture.
  • Network Segmentation: Having multiple subnets allows for better organization and segmentation of resources based on their roles or security requirements.
  • Scalability: Multiple subnets provide room for future growth and expansion of your network infrastructure.

In this case, since we specified "Number of Availability Zones (AZs): 1" in our VPC creation settings, both subnets are likely in the same AZ. However, they can still serve different purposes (e.g., public and private) within that single AZ.

Task 2 - Create a Virtual Machine in the AWS Management Console

Amazon Elastic Compute Cloud (Amazon EC2) is a web service that provides secure, resizable compute capacity in the cloud. It is designed to make web-scale cloud computing easier for developers. Amazon EC2's simple web service interface allows you to obtain and configure capacity with minimal friction.

Key features of Amazon EC2 include:

  • Elastic: You can increase or decrease capacity within minutes, not hours or days.
  • Flexible: You have complete control of your instances. You can select the capacity you need, choose from multiple instance types, operating systems, and software packages.
  • Inexpensive: Amazon EC2 passes on the financial benefits of Amazon's scale. You pay a very low rate for the compute capacity you actually consume.
  • Reliable: Amazon EC2 offers a highly reliable environment where replacement instances can be rapidly and predictably commissioned.
  • Secure: Amazon EC2 works in conjunction with Amazon VPC to provide security and robust networking functionality.

Amazon EC2 is a fundamental component of Amazon's cloud computing platform, allowing businesses of all sizes to run their applications on Amazon's proven computing environment.

  1. From Services, navigate to Compute, then EC2.
  1. Select Launch instance.
  1. Name instance.
  1. Under Application and OS images (Amazon Machine Image), select Windows. Keep t2.micro as the instance type.
  1. For Key pair (login), select Create new key pair.
  1. Name your key pair. Key pair name must be unique. Once named, click Create key pair.
  1. Back on the EC2 creation page, click edit on Network settings.
  1. Select our previously created VPC. In subnet, select the subnet that was created along with our VPC.
  1. Select Enable for Auto-assign public IP.
  1. Everything else remains on default. You may now launch the instance.
  1. Return to the EC2 dashboard page and then select the instance and then Connect.
  1. On the Connect to instance page, select RDP Client.
  1. Scroll down to Get password.
  1. On the Get Windows password page, select Upload private key file. Then navigate to the key pair file we created earlier.
  1. Once the file is uploaded, select Decrypt password.
  1. On the RDP tab, copy the generated password, then click Download remote desktop file.
  1. Click the downloaded remote desktop file in the download pop-up window.
  1. Click connect.
  1. Paste the password that we copied earlier.
  1. Click Yes on the Remote Desktop Connection window.
  1. We have successfully connected to our EC2 instance.
