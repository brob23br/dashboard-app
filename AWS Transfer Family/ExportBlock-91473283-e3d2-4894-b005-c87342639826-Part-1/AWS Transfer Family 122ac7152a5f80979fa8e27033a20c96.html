We will be creating a SFTP server and connecting to it via SSH.

Before we dive into the steps, let's briefly overview the AWS Transfer Family service:

AWS Transfer Family is a fully managed service that enables the secure transfer of files over SFTP, FTPS, and FTP protocols directly into and out of Amazon S3 or Amazon EFS. Key features include:

  • Seamless integration with existing authentication systems
  • Support for multiple protocols (SFTP, FTPS, FTP)
  • Automatic scaling to meet your file transfer demands
  • End-to-end encryption for data in transit and at rest
  • Detailed CloudWatch metrics and logs for monitoring
  • Integration with AWS services like S3, EFS, IAM, and Route 53

This service is particularly useful for organizations that need to maintain compliance with data transfer regulations while modernizing their file transfer workflows. It eliminates the need to manage your own file transfer infrastructure, reducing operational overhead and improving security.

Now, let's proceed with setting up our SFTP server using AWS Transfer Family:

  1. On the AWS Management console, navigate to S3. An empty S3 bucket will be needed which was created in a preceding demo. https://www.notion.so/AWS-Storage-Services-109ac7152a5f8020a126c9ccb894dd40?pvs=4 .
  1. We need to create a role that will be applied to the bucket. This will be done with IAM. Navigate to IAM and then select Roles on the left pane.
  1. Select Create Role
  1. Under Trusted Entity type, select AWS Service. Then under Use Case, select EC2. Click Next
  1. For permissions, search for AmazonS3FullAccess. Click Next
  1. Name the Role to help distinguish what it will be used for. Click Create Role
  1. Once created, search for and select the role
  1. Under the Summary section, select Trust relationships.
  1. Click Edit Trust Policy. Edit the Service key value to “transfer.amazonaws.com”. Click Update policy
  1. From the AWS Management Console, search AWS Transfer Family. We will create an SFTP transfer server. Click Create server.
  1. For Choose Protocols, leave the default of SFTP selected.
  1. For Choose an identity provider, select Service managed.
  1. On Choose your endpoint, choose Publicly accessible, and Custom hostname set to None.
  1. For Choose a domain,, select Amazon S3.
  1. Nothing will be changed on Configure additional details. Scroll down and click Next.
  1. On Review and create, scroll down and create server.
  1. Users need to be added to the server. Select the newly created server and then click Add user.
  1. Add a username and then select the SFTP role we created.
  1. For Home Directory, select the S3 bucket created in step 1.
  1. Next, a public key of the machine that will be used for access should be added to the SSH public key field. A tutorial on how to generate SSH keys can be found here, https://www.notion.so/SSH-keygen-697e52a4f4864a4b89b5be30db3fef5d?pvs=4 . Once your public key is added, click Add.
  1. Once created, on the AWS Transfer Family console, copy the endpoint name. This is the DNS name that will be used to connect to the server.
  1. In a command prompt, run the command sftp -i, add the name of your private key file, and then paste the endpoint name you copied. Run the command. It will ask you if you trust the authenticity of the host. I trust this host, so will be selecting Yes
  1. You will then be prompted for the passphrase for your SSH key file. Once authenticated, you will see SFTP. Running a print working directory command will show that we are in our S3 bucket.
